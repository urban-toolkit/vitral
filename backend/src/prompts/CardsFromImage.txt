    - 'person': a person involved in the design study
    - 'requirement': a need, question, or task to be tackled
    - 'concept': a conceptual reference/definition (e.g., nomenclature, paper title, reference to literature, domain jargon)
    - 'insight': a finding derived after thinking, analysis or discussion 
    - 'activity': activity in the design study (e.g, meeting, user study, brainstorming)
    - 'artifact': a software, digital, or data object (e.g., dataset, jupyter notebook)

Based on the content generate **exactly** one activity card and one or more of the other cards. 

You will recieve a JSON describing the image file as:

{
    name: string,
    content: string
}

- Classify the image according to its role potential role in the design study:

    - Inspiration: the interface of another system for inspiration.
    - Schematics: any diagram, plan, drawing that illustrate connections and functional relationships.
    - Chart: a visualization chart.
    - Sketch: any manual or digital sketch of an image.
    - Note: notes taken.
    - Photograph: a real life photograph (e.g., workshop session).
    - Prototype: a system prototype (e.g., wireframe).
- Use the role classification to base card generation:
    - Interface: generate activity 'Interface implementation' and artifact about the image itself.
    - Inspiration: generate acitivy 'Inspiration collection', an artifact about the image itself, and any requirements or insights that could be useful to inform design.
    - Schematics: generate activity 'Creating schema of [complete with correct name]' and an artifact about the image itself.
    - Chart: generate activity 'Data analysis' and an artifact about the image itself.
    - Sketch: generate activity 'Sketching' and an artifact about the image itself.
    - Note: generate activity according to content of note (e.g., brainstorming, meeting, user study session) and any other relevant cards according to the note content.
    - Photograph: generate activity according to content of (e.g., workshop, field research, reference collection), and artifact about the image itself.
    - Prototype: generate activity 'Prototyping' and an artifact about the image itself.

Output a list of cards and connections in the following JSON format:

{
    role: string,
    "cards": [
        {
            entity: string,
            title: string,
            description: string 
        }
    ],
    "connections": [
        {
            source: number,
